auto grid(this T)(int column, int row, int outerPadding = 0, int innerPadding = 0, int columnSpan = 1, int rowSpan = 1, string sticky = "");

Geometry method for placing this widget inside its parent using an imaginary grid. Somewhat more direct and intuitive than pack. Choose grid for tabular layouts, and when there's no good reason to choose something else.

If a widget's cell is larger than its default dimensions, the sticky parameter may be used to position (or stretch) the widget within its cell. The sticky argument is a string that contains zero or more of the characters n, s, e or w. Each letter refers to a side (north, south, east, or west) that the widget will 'stick' to. If both n and s (or e and w) are specified, the slave will be stretched to fill the entire height (or width) of its cell. The sticky parameter subsumes the combination of anchor and fill that is used by pack. The default is an empty string, which causes the widget to be centered in its cell, at its default size. The anchorposition enum may be of use here but doesn't provide all combinations.

Parameters

int column The column in which to place this widget.
int row The row in which to place this widget.
int outerPadding The amound of padding to add around the widget.
int innerPadding The amound of padding to add inside the widget.
int columnSpan The amount of column this widget should span across.
int rowSpan The amount of rows this widget should span across.
string sticky Which edges of the cell the widget should touch. See note above.

Return Value

This widget to aid method chaining.

Caveats:

In order for a gridded UI to be fully dynamic to expand and contract when resizing elements, you must to assign a weight to at least one column and row using configureGeometryColumn and configureGeometryRow respectively.

tkd.widget.widget.GeometryBorderMode

auto place(this T)(double relativeXPos, double relativeYPos, double relativeWidth, double relativeHeight, string anchor = AnchorPosition.northWest, string borderMode = GeometryBorderMode.inside);

Geometry method for placing this widget inside its parent using relative positioning. In this case the position and size is specified as a floating-point number between 0.0 and 1.0 relative to the height of the parent. 0.5 means the widget will be half as high as the parent and 1.0 means the widget will have the same height as the parent, and so on.

Parameters

double relativeXPos The relative horizontal position of the widget inside its parent.
double relativeYPos The relative vertical position of the widget inside its parent.
double relativeWidth The relative width of the widget.
double relativeHeight The relative height of the widget.
string anchor The anchor position of the widget inside its parent.
string borderMode How the widget interacts with the parent's border.

enum GeometrySide: string;

Side values for widget geometry layout.

left

Set geometry to the left.

right

Set geometry to the right.

top

Set geometry to the top.

bottom

Set geometry to the bottom.

enum GeometryFill: string;

Fill values for widget geometry layout.

none

No filling.

x

Fill the available horizontal space.

y

Fill the available vertical space.

both

Fill all available space.

enum GeometryBorderMode: string;

Interaction modes for parent borders.

inside

Take into consideration the border when placing widgets.

outside

Don't take consideration the border when placing widgets.

ignore

Ignore borders.

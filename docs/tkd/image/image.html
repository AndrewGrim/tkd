class Image: tkd.element.element.Element;

The image base class.

this();

Construct the image.

auto embedBase64Data(string filename, this T)();

This method embeds the image as a base64 encoded string into the application at compile-time. The path to the image must be passed to the compiler using the -J switch.

Parameters

filename The filename to read the data from.

Return Value

This image to aid method chaining.

auto blank(this T)();

Clears the image of all pixel data and effectively makes it transparent.

Return Value

This image to aid method chaining.

auto setData(this T)(string data);

Specifies the contents of the image as a string. The string should contain binary data or, for some formats, base64-encoded data (this is currently guaranteed to be supported for PNG and GIF images). A set file takes precedence over the setting of data.

Parameters

string data The image data.

Return Value

This image to aid method chaining.

string getData();

Get the image data.

Return Value

Base64 encoded image data.

auto setFormat(this T)(string format);

Set the image format. Once set the image only accepts files or data in this format.

Parameters

string format A valid image format.

Return Value

This image to aid method chaining.

See Also

tkd.image.imageformat for supported formats.

Get the image format.

Return Value

The image format.

See Also

tkd.image.imageformat for returned formats.

auto setPalette(this T)(string palette);

Set the image palette. This setting set how many levels of color/gray are used.

Sample Palettes:

  • "16" = 16 levels of gray
  • "16/16/16" = 16 levels of RGB

Parameters

string palette A string describing the palette to use.

Return Value

This image to aid method chaining.

string getPalette();

Get the current palette.

Return Value

A string describing the palette.

void destroy();

Destroy this image.

Caveats:

Once an image is destroyed it can no longer be referenced in your code or a segmentation fault will occur and potentially crash your program.
